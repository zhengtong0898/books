# 什么是工厂方法(Factory Method)?
A Factory is an object for creating other objects.    
用于创建对象的对象(函数)被称为工厂方法.   


&nbsp;  

# 无模式编码
创建一个对象最了当的方式就是直接创建, 但这种方式并不适合所有场景. **比如说:**   
当创建对象的场景是根据不同条件来创建时, 每种场景对应的其实就是一个条件句, 而每个条件句下面都对应一个创建对象的代码.   

### 优点
适合在写小工具(场景规模单一、小)的时候使用, 因为直观，容易维护, 编码效率高.

### 缺点
场景条件复杂的情况下, 无模式编码的代码量通常是最多的, 而且写出来的代码通常都是巨量的条件句, 也许很直观, 也许编码效率高, 但是一定不容易扩展(维护).


&nbsp; 

# 模式编码
无模式编码就像是一个函数不提供参数, 硬编码的感觉, 有多少种场景就写多少个函数.  
而工厂模式就像是一个函数提供了一个参数(类型), 函数体内针对这个参数预定义了所有可能的映射.

### 优点
容易扩展(维护).

### 缺点
1. 代码阅读不直观   
2. 编码效率前期低，后期高.  

&nbsp;

# 总结
无模式编码属于横向排铺,  
模式编码属于结构化排铺或层级排铺.

结构化指的是: 预定义映射
